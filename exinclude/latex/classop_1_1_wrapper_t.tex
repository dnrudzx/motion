class op\+::\+Wrapper\+T$<$ T\+Datum, T\+Datums, T\+Datums\+S\+P, T\+Worker $>$}

\mbox{\hyperlink{classop_1_1_wrapper_t}{WrapperT}}\+: Open\+Pose all-\/in-\/one wrapper template class. Simplified into Wrapper for \mbox{\hyperlink{classop_1_1_wrapper_t}{WrapperT}}$<$std\+::vector$<$\+Datum$>$$>$ \mbox{\hyperlink{classop_1_1_wrapper_t}{WrapperT}} allows the user to set up the input (video, webcam, custom input, etc.), pose, face and/or hands estimation and rendering, and output (integrated small G\+UI, custom output, etc.).

This function can be used in 2 ways\+:
\begin{DoxyItemize}
\item Synchronous mode\+: call the full constructor with your desired input and output workers.
\item Asynchronous mode\+: call the empty constructor \mbox{\hyperlink{classop_1_1_wrapper_t_a94151754dddc2a37044aea26b9dac6c7}{Wrapper\+T()}} + use the emplace and pop functions to push the original frames and retrieve the processed ones.
\item Mix of them\+:
\begin{DoxyItemize}
\item Synchronous input + asynchronous output\+: call the constructor \mbox{\hyperlink{classop_1_1_wrapper_t}{WrapperT}}(\mbox{\hyperlink{namespaceop_a3593e2d53bec533f0048ef3973eebd36a2fe4167817733fec8e6ba1afddf78f1b}{Thread\+Manager\+Mode\+::\+Synchronous}}, workers\+Input, \{\}, true)
\item Asynchronous input + synchronous output\+: call the constructor Wrapper\+T(\+Thread\+Manager\+Mode\+::\+Synchronous, nullptr, workers\+Output, irrelevant\+Boolean, true) 
\end{DoxyItemize}
\end{DoxyItemize}
