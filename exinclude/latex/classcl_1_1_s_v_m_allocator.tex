\subsubsection*{template$<$typename T, class S\+V\+M\+Trait$>$\newline
class cl\+::\+S\+V\+M\+Allocator$<$ T, S\+V\+M\+Trait $>$}

S\+T\+L-\/like allocator class for managing S\+VM objects provided for convenience.

Note that while this behaves like an allocator for the purposes of constructing vectors and similar objects, care must be taken when using with smart pointers. The allocator should not be used to construct a unique\+\_\+ptr if we are using coarse-\/grained S\+VM mode because the coarse-\/grained management behaviour would behave incorrectly with respect to reference counting.

Instead the allocator embeds a Deleter which may be used with unique\+\_\+ptr and is used with the allocate\+\_\+shared and allocate\+\_\+ptr supplied operations.
